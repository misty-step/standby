# Design Acceptance

Before closing a UI change, verify:

- Capture state is visible as a quiet status indicator, not as a dashboard block.
- The default view is the Action Stream: suggested work, running jobs, completed
  outputs, failures, and no-card events in one readable feed.
- Suggested work and meeting-aware chat can be combined in one thread.
- Running work and completed outputs are visible in the stream; transcript/source
  detail stays reachable through compact counts, tabs, shelves, drawers, or
  focused detail views.
- Transcript detail is tucked behind a source drawer, citation inspector, or
  explicit transcript control.
- Suggested action cards show cited transcript evidence and an editable prompt.
- Approving a suggestion creates visible job state and eventually a receipt or
  failure reason.
- Navigation switches meaningful agent-work panels without forcing all panels
  onto the default screen.
- Audio status names microphone and call/system lanes separately.
- The layout remains usable at desktop and mobile widths without text overlap.

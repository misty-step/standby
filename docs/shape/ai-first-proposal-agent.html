BACKLOG SOURCE: AI-FIRST PIVOT

Replace keyword proposal detection with a model-native proposal agent.

Standby should understand live meeting context, propose useful action cards, accept operator nudges, and preserve deterministic approval. The first slice builds the model boundary and quality oracle before expanding workers or diarization.

CHOSEN DESIGN

Typed ProposalAgent boundary with recorded CI provider and opt-in live model provider. The heuristic engine has been removed from source.

PROOF SURFACE

./scripts/verify-model-proposals.sh, ./scripts/verify-manual-proposal-request.sh, full gate, and opt-in live model smoke.

STANDARD

Model owns semantic judgment. Rust owns authority, logs, policy, approval, and schema.

ACCEPTANCE

Paraphrased asks produce grounded cards; vague or negated chatter produces no card.

VERIFY

Fixture grader, route replay, full gate, and permission-gated live provider evidence.

STOP IF

Raw audio must be committed, workers run without approval, or cloud model calls become default without explicit config.

CURRENT STATE

engine.rs now uses model-response fixtures or an opt-in live model provider, then Rust validates schema, evidence spans, and approval policy. Worker status is visible; network/model workers remain gated.

CHANGE SHAPE

Add a proposal-agent interface, recorded model fixtures, one live model adapter, model metadata events, no-proposal states, and UI source/confidence display.

NON-GOALS

No default network worker, no autonomous external sends, no named speaker identity, no full UI redesign.

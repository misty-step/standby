STANDBY / 004

Tool-capable model worker boundary

BACKLOG 004 / EXECUTION CONTRACT

Add an opt-in OMP research worker without making meetings an exfiltration path.

Standby will gain a real model/tool worker profile for approved jobs, but the default remains the network-denied local worker. The worker can start only after global opt-in, deterministic approval, per-job network consent, prompt redaction, isolated home setup, and profile-policy verification.

CHOSEN DESIGN

omp-research launches omp -p with GLM 5.2 by default, a per-job home/session directory, forwarded provider auth keys only, and an explicit tool allowlist.

QUALITY SYSTEM

High-stakes boundary: live route replay, structural tests, sandbox negative test, evidence packet, and fresh-context security/architecture critic.

STANDARDS

Rust owns policy, approval, eventing, sandboxing, profile selection, and receipts. CLI flags are defense in depth, not the security boundary.

ACCEPTANCE

omp-research is ignored without global opt-in; with opt-in it still needs per-job consent and fails visibly when auth is absent.

VERIFY

scripts/verify-model-worker-boundary.sh, scripts/verify-worker-sandbox.sh, targeted Rust tests, and full ./scripts/verify.sh.

STOP IF

OMP cannot run noninteractively, needs user-home credential reads, requires write access outside scratch, or cannot disable unapproved tools.

CURRENT STATE

008 merged operator auth, per-job network consent, and prompt redaction. Existing cloud profiles are network-gated but not the desired OMP/GLM worker.

CHANGE SHAPE

Add profile metadata for isolated home, static env, and allowed tools; implement WorkerProfile::omp_research(); add executable verifier receipts.

PROOF SURFACE

Evidence lands in docs/evidence/operator-action-control/model-worker-boundary/ with projections, receipts, and a JSON verdict.

NON-GOALS

No default network worker. No repo mutation profile. No MCP marketplace. No browser/email/Slack/send tools. No weakening of the local worker gate.

CRITIC FOCUS

Can transcript text reach an unapproved tool? Can the worker read user secrets? Can missing auth hang or disappear? Can profile fallback silently overclaim OMP execution?

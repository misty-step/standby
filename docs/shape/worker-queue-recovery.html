STANDBY / 006

worker queue recovery

PREMISE / backlog.d/006-recover-worker-queue-after-daemon-restart.md

Approved OpenCode jobs survive daemon restart.

Chosen design: replay the append-only event log on startup and enqueue any latest job state that is still queued or running. Do not add a second durable queue table until the event log stops being sufficient.

QUALITY BAR

One ledger owns truth. Approval path stays unchanged. Recovery uses the same worker loop as live approval. Completed, failed, and canceled jobs are never rerun.

ALTERNATIVES

Queue table: rejected as duplicate truth. Manual retry: rejected as operator-hostile. Supervisor process: deferred as larger runtime shape.

ORACLE

scripts/verify-worker-recovery.sh kills a daemon mid-job, restarts it, observes terminal job state, restarts again, and proves the completed job did not rerun.

STOP IF

Recovery requires changing approval semantics, adding mutable external worker actions, or replay cannot distinguish terminal jobs from active jobs.

IMPLEMENTATION

1 core

Add `EventStore::recoverable_jobs()` over job events ordered by insertion.

Unit tests cover queued/running returned and terminal skipped.

2 daemon

After spawning the worker loop, enqueue recoverable jobs from the opened store.

No new route or user setting.

3 proof

Add fake OpenCode wait marker and restart smoke.

Evidence packet under `docs/evidence/operator-action-control/worker-recovery/`.

4 gate

Wire focused verifier into `./scripts/verify.sh` and hosted CI if fast enough.
